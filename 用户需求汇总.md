# 瑞界物业（SG Properties）— 用户需求汇总

> 整理日期：2026-05-24 | 涵盖全部对话记录、头脑风暴、UI评审、功能迭代

---

## 一、系统愿景与定位

### 1.1 项目背景

- 项目位于**越南·北宁省·嘉平**（Bac Ninh, Vietnam）的工业园区
- 品牌名：**瑞界物业**（SG Properties）
- 系统定位：物业承接查验 + 日常巡检 + 设备维护的**综合管理系统**
- 优先**电脑端**使用，后续再考虑手机端/小程序

### 1.2 核心业务流程

```
管理员建项目 → 配置楼栋房源 → 从参考模板保存为我的表单 → 
打印空白表去现场 → 纸质填写 → 回系统录入 → 提交锁定 → 永久存档
```

### 1.3 设计理念

- **简洁实用**：去掉复杂的授权机制，方便一线员工使用
- **纸质+电子双存档**：电子记录与纸质签字表并行
- **一次性承接查验 + 反复日常巡检**：两套数据分开管理，但操作逻辑统一

## 二、用户角色与权限

### 2.1 三级角色

| 角色 | 用户名格式 | 权限范围 |
|------|-----------|----------|
| **系统管理员**（admin） | 自定义 | 全部权限：建项目、管用户、编辑模板 |
| **物业经理**（manager） | SG + 3位数字 | 项目配置、仪表盘、查看全部记录 |
| **基层员工**（employee） | SG + 3位数字 | 填自己的记录、看自己的表单、打印 |

### 2.2 权限细则

- 员工只能看到和编辑**自己创建**的记录
- 经理/管理员可看到**所有人**的记录
- 所有用户可看到**全部参考表单**（无需授权机制）
- 经理/管理员可访问仪表盘和项目配置
- 只有管理员可访问用户管理页面（`/admin/users`）

### 2.3 记录提交与锁定

| 操作 | 创建者 | 经理/管理员 |
|------|:------:|:----------:|
| 保存草稿 | ✓ | ✓ |
| 提交锁定 | ✓ | ✓ |
| 查看已提交记录 | 仅自己 | 全部 |
| 修改已提交记录 | ✗ | ✗ |
| 删除已提交记录 | ✗ | ✗ |

> 提交是不可逆操作，提交后任何人无法修改或删除。

## 四、功能模块详细需求

### 4.1 我的表单

**用途**：员工创建和管理自己的查验表单，以及打印空白表

**功能**：
- 创建空白表单（手工添加检查项）
- 从参考模板保存为"我的表单"（命名为：**原模板名 - 用户名**）
- 编辑表单：添加/删除/修改检查项、批量编辑
- 打印空白表单（跳转打印预览）
- 删除表单（级联删除关联记录）
- **不在"我的表单"中录入检查结果**，录入统一在"日常巡检/承接查验"中进行

**布局**：2 列卡片网格，每张卡片显示编号、标题、条目数、创建者、日期

### 4.2 日常巡检

**用途**：日常巡查记录的填写和管理

**功能**：
- 新增记录：只能从**我的表单**中选择（不能从参考模板选）
- 选择位置：楼栋 → 房号 → 自由文本位置
- 逐项录入：合格/不合格（需填问题描述）/免检
- 每项可上传照片（最多 6 张，每张 ≤10MB）
- 填写查验意见
- 保存记录（草稿，可继续编辑）
- 提交记录（永久锁定）
- 筛选：按模板、楼栋、状态
- 已提交记录不可删除

### 4.3 承接查验

**用途**：新项目交付时的承接查验记录

**功能**：
- 操作逻辑与日常巡检完全一致
- 记录类型标记为 `acceptance`
- 与日常巡检**分列表展示**
- 通常一次性完成，存档后不再修改
- 承接查验的历史记录永久保留，与后续日常巡查记录并列

### 4.4 设备档案

**用途**：设备台账 + 维护计划 + 技术资料管理

**三个子标签页**：

#### 基本信息
- 设备名称、类别（配电/消防/电梯/给排水/暖通/弱电/其他）
- 型号、编号、状态（正常/保养中/待修/已报废）
- 安装日期、位置、备注
- 经理/管理员可编辑

#### 维护计划
- **手工添加**维护计划（日期 + 内容描述）
- 维护记录表格：计划日期、内容、状态（待执行/已完成/已跳过/已逾期）、完成日期、备注、照片
- 各状态用不同颜色标记
- 编辑弹窗：更新状态、完成日期、备注、上传照片
- 维护完成后可提交锁定
- 统计行：X 项 · Y 已完成 · Z 待执行
- 维护状态指示灯：🟢 7天+ / 🟡 7天内 / 🔴 已逾期

#### 说明书
- 上传设备说明书、操作手册（PDF/Word/Excel/图片）
- 查看、下载、删除

**布局**：2 列卡片网格，筛选栏支持类别和状态过滤

### 4.5 参考表单

**用途**：系统预设的标准化查验模板库

**分类**：
- **A. 设备机房查验**（10 张）：变配电室、发电机房、水泵房、消防控制室、电梯机房、冷水机组、通风排烟、消防水系统、消防报警、电梯轿厢
- **B. 公共区域及设施查验**（8 张）：监控安防、道路停车场、绿化景观、围墙大门、物业用房、给排水外网、弱电智能化、防雷接地
- **C. 室内专有部分查验**（4 张）：室内土建、室内给排水、室内电气、可租房源
- **D. 资料与物品移交**（4 张）：技术资料移交、钥匙门禁移交、设备台账汇总、问题汇总整改

**功能**：
- 员工可查看全部模板
- 点击"查看"预览模板内容
- 从预览页"保存为我的表单"
- **不能从参考表单直接打印**
- 管理员可新建/编辑模板

### 4.6 打印

**入口**：我的表单 → 打印按钮；或打印选择页面批量打印

**打印规则**：
- 强制两页 A4（内容少的第二页留空白表格供手写）
- 第一页：22 行数据 + 项目信息表 + 查验意见 + 签名栏
- 第二页：26 行数据 + 查验意见 + 签名栏
- 抬头**只在第一页**显示
- 所有竖线对齐（`table-layout: fixed`，统一列宽 4%/15%/45%/36%）
- 照片单独分页

**打印方式**：浏览器打印（Ctrl+P / Cmd+P）

**命名规则**：抬头显示"瑞界物业 查验记录表"

### 4.7 仪表盘

**用途**：项目统计分析（经理/管理员）

**功能**：
- 时间范围选择：起止月份，快捷按钮（近3月/近6月/近1年）
- 概览卡片：总记录数、不合格项、已完成、待处理
- 月度趋势柱状图
- 按模板统计（完成率）
- 按人员统计（完成率）
- 最近提交记录列表

### 4.8 项目配置

**用途**：项目管理（经理/管理员）

**四个标签页**：
1. **项目信息**：名称、类型、地址、面积、交接日期、开发商、经理信息
2. **楼栋管理**：添加/删除楼栋
3. **房源管理**：选择楼栋、添加房号、面积
4. **成员管理**：添加成员（SG自动编号 + 随机密码）、角色分配、重置密码、移除成员

### 4.9 用户管理

**用途**：系统用户账户管理（仅管理员）

- 查看全部用户
- 创建用户（用户名、密码、姓名、角色）
- 删除用户（不能删自己）

## 五、UI/UX 设计规范

### 5.1 核心原则

> **统一性、一致性、美观性**

### 5.2 布局规范

- 所有内容宽度统一 `max-width: 800px`
- 列表统一 2 列卡片网格
- 左侧导航固定，右侧内容滚动
- 相同功能的元素外观和位置必须一致

### 5.3 按钮规范

| 场景 | 样式 |
|------|------|
| 页面头部操作按钮 | `btn btn-sm` |
| 主要提交按钮 | `btn` |
| 取消/返回按钮 | `btn btn-outline` |
| 危险操作（删除） | `btn btn-sm btn-danger-outline` |
| 小按钮（编辑、查看） | `btn btn-sm btn-outline` |

### 5.4 卡片规范

所有列表页面的卡片必须使用完全相同的 HTML 骨架：
- 左侧：`flex-1 min-w-0` — badge + 标题（衬线体 16px/600）+ 信息行
- 右侧：操作按钮区 `flex gap-8 shrink-0 items-center`
- 内边距默认 20px

### 5.5 模态框规范

| 类型 | class | 宽度 |
|------|-------|------|
| 默认 | `modal-card` | max-width: 800px |
| 简单表单 | `modal-card-sm` | 440px |
| 复杂表单 | `modal-card-lg` | 700px |

### 5.6 颜色与字体

- 主色：藏蓝（`#1e3a5f`）
- 衬线字体（Noto Serif SC）用于标题
- 只使用 CSS 变量，禁止硬编码颜色值
- 登录页背景：明亮的工业厂房/园区照片，铺满全屏

### 5.7 工具类优先

禁止 inline style，必须使用全局工具类：
`text-truncate`、`shrink-0`、`min-w-0`、`flex-1`、`text-center`、`text-sm`、`text-secondary`、`mb-4/8/12/16`、`gap-4/8/12/16`

## 六、数据与安全

### 6.1 数据库

- SQLite（`backend/data.db`）
- 照片存储在 `backend/uploads/` 目录
- 数据库和照片目录需**同步备份**

### 6.2 数据安全规则

- 记录提交后前端+后端双重锁定
- 记录创建时检查项名称当场固定（数据快照），后续模板修改不影响历史记录
- 删除表单级联删除关联记录
- 每个表单的每次检查创建新记录，互不覆盖

### 6.3 部署方向

- 当前：本地开发环境（`localhost:5173` + `localhost:3002`）
- 目标：腾讯云部署（已申请域名）
- 备份：定期复制数据库文件，可备份到网络云盘

## 九、用户确认的设计决策

1. 去掉项目列表页，单项目系统，登录后直接进入"我的表单"
2. 左侧导航分三个区：工作台、资源、账户
3. 参考表单独立标签，不与我的表单合并
4. 检查记录不允许修改表单结构，只能录入结果
5. 录入时只能从"我的表单"选择，不能从参考模板选
6. 表单命名规则：参考表单名 + 用户名
7. 所有打印强制两页 A4，不足两页的第二页保留空表格
8. 列宽统一：序号 4% + 检查项目 15% + 检查标准 45% + 查验结果 36%
9. 五个标签页列表显示格式完全一致（2列卡片网格）
10. 内容区宽度统一 800px
11. 设备档案不设"周期"功能，手工添加维护计划
12. 承接查验放在设备档案下方（左侧导航）
13. D类模板保留在参考表单中
14. 登录页背景用明亮、阳光的工业厂房图片
15. 系统名称从"物业承接查验系统"改为"瑞界物业"
16. 英文名：SG Properties
